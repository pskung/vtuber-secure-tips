# VTuber Secure Tips (PDPA-Compliant Model)

Highly resilient, modular, stateless backend gateway designed on Google Apps Script (GAS) Serverless context. Facilitates and triggers dynamic PromptPay QR code generation and live stream notification alerts anonymized under Thailand's PDPA rules.

---

## 🚀 Key Security Features
1. **PDPA/GDPR Compliance:** Discards real bank names and account strings. Broadcasts only display nicknames.
2. **Aggregated Self-Healing Circuit Breaker:** Dynamically drops broken alert endpoints without affecting payment records [1.1.3].
3. **Anti-Automation Throttling:** Multi-layered Proof-of-Work client check combined with dynamic global spike-rate queue limits.
4. **Strict Nonce CSP:** Zero 'unsafe-inline' elements. Secure dynamic event listeners mapped under active nonce checks.
5. **30-Day Automated Data Purge:** Automated daily time-driven cron cleanses donor history older than 30 days.

# VTuber Secure Tips (PDPA-Compliant Model)

โครงสร้างระบบเชื่อมต่อและแปลงรหัสยอดบริจาคเข้าสู่การแจ้งเตือนสตรีมเมอร์ ทำงานบนสถาปัตยกรรมไร้เซิร์ฟเวอร์ (Serverless) และไม่มีการเก็บข้อมูลอ่อนไหวถาวรบนระบบ (Stateless) พัฒนาขึ้นด้วยบริการคลาวด์ขนาดเล็กของ Google Apps Script (GAS) เพื่อส่งต่อลายเซ็นยอดจ่ายเงิน PromptPay แบบไม่ผ่านผู้รับภายนอก พร้อมมาตรฐานความสอดคล้องต่อนโยบายคุ้มครองข้อมูลส่วนบุคคลของไทย (PDPA) อย่างสมบูรณ์

---

## 🚀 จุดเด่นทางระบบความปลอดภัยและฟีเจอร์เด่น
1. **สอดคล้องกฎหมาย PDPA/GDPR:** ไม่เก็บข้อมูลชื่อบัญชีจริงหรือเลขบัญชีธนาคาร แสดงผลเฉพาะชื่อเล่นที่ผู้ใช้กรอกเข้ามาเท่านั้น
2. **ระบบสับคัตเอาต์เยียวยาตัวเองอัตโนมัติ (Circuit Breaker):** หลีกเลี่ยงหน้าสตรีมค้างหรือล่ม โดยระบบจะตัดสัญญาณฝั่งหน้าจอเตือน Streamlabs อัตโนมัติหากพบสัญญาณติดขัด แต่กระบวนการชำระเงินยังทำงานปกติ
3. **ระบบสกัดบ็อตก่อกวน (Anti-Automation Throttling):** กลไกบังคับฝั่งหน้าเว็บให้คำนวณถอดโจทย์พรูฟออฟเวิร์ก (Proof-of-Work) ร่วมกับมาตรการบล็อกยอดคำขอพุ่งกระฉูดระดับ Global Spike
4. **นโยบายความมั่นคงปลอดภัยสคริปต์ (Strict Nonce CSP):** ป้องกันช่องทางลักลอบโจมตีหรือเปลี่ยนโครงสร้างโค้ดหน้าเว็บบนเบราว์เซอร์ด้วยสัญชาตลักษณ์ป้องกันสคริปต์แอบแฝง
5. **ระบบทุบทำลายประวัติธุรกรรมประจำวัน (30-Day Data Purging):** ติดตั้งระบบทำงานนัดแนะตรวจสอบประวัติในฐานข้อมูล Firebase ทุก 24 ชั่วโมงเพื่อลบข้อมูลที่อายุเกิน 30 วันทิ้งตามกฎหมายกำหนด

### ขั้นตอนที่ 2.1: การสร้างและวางโค้ดใน Google Sheets
เราจะติดตั้งระบบผ่าน "สคริปต์ที่ผูกกับตารางคำนวณ" (Container-Bound Script) ซึ่งสะดวกที่สุดและสามารถโยกย้ายไฟล์ได้ง่ายค่ะ

1. เปิดเบราว์เซอร์ไปที่ [Google Sheets](https://sheets.google.com)
2. สร้าง **ตารางคำนวณเปล่า (Blank Spreadsheet)** ขึ้นมา 1 ไฟล์ และตั้งชื่อไฟล์ตามต้องการ เช่น `VTuber Secure Tips DB`
3. ที่แถบเมนูด้านบน คลิกเลือก **ส่วนขยาย (Extensions)** -> **Apps Script**
4. คุณจะพบหน้าต่างตัวเขียนโค้ดสีเข้ม (Apps Script Editor) และมีไฟล์เริ่มต้นชื่อ `Code.gs` ปรากฏอยู่
5. ให้ล้างโค้ดตัวอย่างใน `Code.gs` ออกให้หมด จากนั้นทำตามขั้นตอนดังนี้เพื่อเพิ่มไฟล์ให้ครบถ้วน:
   * **การเพิ่มไฟล์ประเภทสคริปต์ (.gs):** คลิกที่ปุ่มเครื่องหมายบวก `+` ข้างหัวข้อ "ไฟล์" -> เลือก **สคริปต์ (Script)** -> ตั้งชื่อตามไฟล์ระบบ (เช่น `Config`, `CircuitBreaker` เป็นต้น) แล้วนำโค้ดในแต่ละไฟล์จากคลัง GitHub ที่อาคาริจัดเตรียมไว้ให้ก่อนหน้านี้ไปวางลงไปค่ะ
   * **การเพิ่มไฟล์ประเภทเว็บหน้าต่าง (.html):** คลิกที่ปุ่มเครื่องหมายบวก `+` ข้างหัวข้อ "ไฟล์" -> เลือก **HTML** -> ตั้งชื่อตามไฟล์ระบบ (เช่น `Index`, `GrapesJSForm` และ `GrapesJSQR` **โดยไม่ต้องพิมพ์นามสกุล .html**) จากนั้นคัดลอกโค้ด HTML ทั้งหมดไปวางแทนที่ค่ะ
   * ตรวจสอบว่ามีไฟล์ทั้งหมดครบถ้วนตามชื่อโครงสร้างและบันทึกผล (กดรูปแผ่นดิสก์ หรือกด `Ctrl + S` / `Cmd + S` บนคีย์บอร์ด)

### ขั้นตอนที่ 2.2: การเปิดใช้และตั้งค่าฐานข้อมูล Firebase Realtime Database
ระบบนี้ใช้ Firebase สำหรับเก็บข้อมูลประวัติอย่างปลอดภัยแบบสเตตเลส (Stateless) และจะลบข้อมูลที่เก่ากว่า 30 วันโดยอัตโนมัติค่ะ

1. ไปที่หน้าแรกของ [Firebase Console](https://console.firebase.google.com/) แล้วเข้าสู่ระบบด้วยบัญชี Google ของคุณ
2. คลิกปุ่ม **Add project (เพิ่มโครงการ)** -> ตั้งชื่อโครงการของคุณ เช่น `vtuber-tips-gateway` -> กด **Continue (ถัดไป)** (ในส่วนของ Google Analytics แนะนำให้กดปิดสวิตช์ เพื่อลดความยุ่งยากในขั้นตอนเริ่มต้นค่ะ) -> กด **Create project (สร้างโครงการ)**
3. เมื่อสร้างโครงการเสร็จแล้ว ที่แถบเมนูด้านซ้ายคลิกเลือกหัวข้อ **Build (สร้าง)** -> **Realtime Database**
4. คลิกปุ่ม **Create Database (สร้างฐานข้อมูล)**:
   * **Location (ตำแหน่งพื้นที่บริการ):** แนะนำให้เลือกเป็น **Singapore (asia-southeast1)** เนื่องจากเป็นเซิร์ฟเวอร์คลาวด์ที่ตั้งอยู่ใกล้กับประเทศไทยมากที่สุด ซึ่งจะช่วยลดค่าความหน่วง (Latency) ในการตอบสนองได้เป็นอย่างดีค่ะ
   * **Security Rules (กฎความปลอดภัยขั้นต้น):** เลือกเป็น **Start in locked mode (เริ่มต้นในโหมดล็อก)** แล้วกด **Enable (เปิดใช้งาน)**
5. เมื่อเข้ามาที่หน้าจัดการฐานข้อมูลแล้ว ให้ไปที่แท็บ **Rules (กฎ)** ด้านบน จากนั้นคัดลอกโค้ดจากไฟล์ `firebase/database-rules.json` ไปวางทับค่าเดิมทั้งหมด แล้วกดปุ่ม **Publish (เผยแพร่)**
6. สลับกลับมาที่แท็บ **Data (ข้อมูล)** คุณจะพบลิงก์ URL ของฐานข้อมูลด้านบนสุด (เช่น `https://xxxx.asia-southeast1.firebasedatabase.app/`) ให้คัดลอกลิงก์นี้เก็บไว้เพื่อใช้งานในขั้นตอนถัดไปค่ะ
7. **วิธีการนำรหัสกุญแจฐานข้อมูล (Database Secret) มาใช้งาน:**
   * ที่แถบเมนูด้านซ้าย คลิกไอคอนฟันเฟือง ⚙️ ข้างคำว่า "Project Overview" -> เลือก **Project settings (การตั้งค่าโครงการ)**
   * คลิกแท็บ **Service accounts (บัญชีบริการ)** ด้านบน
   * คลิกหัวข้อ **Database secrets (รหัสลับฐานข้อมูล)**
   * กดปุ่ม **Show (แสดง)** ที่แถบรหัสลับของโครงการของคุณ แล้วคัดลอกคีย์ชุดนี้เก็บไว้ใช้งานในส่วนถัดไปค่ะ

### ขั้นตอนที่ 2.3: การตั้งค่าบัญชีรับเงิน PromptPay บน Xendit
Xendit จะทำหน้าที่ออกใบแจ้งยอดสแกน QR Code แบบแปรผันยอดเงินสนับสนุน และตรวจสอบว่าผู้สแกนชำระเงินจริงหรือไม่ให้โดยอัตโนมัติค่ะ

1. สมัครใช้งานหรือเข้าสู่ระบบที่ [Xendit Dashboard](https://dashboard.xendit.co/)
2. ตรวจดูสวิตช์ที่มุมบนซ้ายของหน้าจอแผงควบคุมหลัก:
   * **Test Mode (โหมดทดลอง):** แนะนำให้เปิดค้างไว้ที่สถานะนี้ก่อน เพื่อรันจำลองการชำระเงินโดยไม่ต้องจ่ายเงินจริง ๆ ค่ะ
   * **Live Mode (โหมดใช้งานจริง):** สลับมาที่โหมดนี้เมื่อคุณทำการทดสอบระบบเสร็จสิ้นและต้องการเปิดรับเงินบริจาคจริงเข้าสู่บัญชีธนาคารของคุณ
3. ไปที่แถบเมนูด้านซ้าย เลือก **Settings (การตั้งค่า)** -> เลื่อนลงไปที่แถบ *Developers* แล้วคลิกหัวข้อ **API Keys**
4. คลิกปุ่ม **Generate Secret Key**:
   * ตั้งชื่อคีย์ตามต้องการ เช่น `VTuberTipsKey`
   * ที่หัวข้อสิทธิ์การใช้งาน (Permissions): ตั้งค่าสิทธิ์ **Money-in Products** เป็นแบบ **Write (เขียน)** ส่วนหัวข้ออื่น ๆ ทั้งหมดให้ตั้งค่าเป็น **None (ไม่มี)**
   * กดปุ่มสร้างระบบ ยืนยันรหัสผ่านเพื่อความปลอดภัย แล้ว**คัดลอกรหัส API Key ที่ขึ้นต้นด้วย `xnd_...` ทันทีและเก็บไว้ในที่ปลอดภัย** (รหัสนี้จะปรากฏให้คัดลอกได้เพียงครั้งเดียวเท่านั้นค่ะ)
5. ไปที่แถบเมนู **Settings (การตั้งค่า)** อีกครั้ง -> เลื่อนไปที่แถบ *Developers* แล้วคลิกหัวข้อ **Webhooks**
6. เลื่อนลงไปด้านล่างสุด คุณจะพบคีย์ข้อความยาว ๆ ในหัวข้อ **Webhook Verification Token** ให้ทำการกดคัดลอกชุดรหัสนี้เก็บไว้เพื่อใช้ป้อนลงระบบหลังบ้านค่ะ

### ขั้นตอนที่ 2.4: การระบุค่าความปลอดภัยผ่านหน้าจอ Google Sheets Sidebar
ตอนนี้เรามีคีย์สำคัญทั้งหมดเรียบร้อยแล้วค่ะ เราจะนำไปบันทึกลงระบบผ่านกล่องตั้งค่าภายในตารางคำนวณในคลิกเดียวโดยไม่ต้องพิมพ์โค้ดเองค่ะ

1. สลับกลับมาที่หน้าตารางคำนวณ Google Sheets ของคุณที่เปิดค้างไว้จากขั้นตอนที่ 2.1
2. ให้ทำการรีเฟรชหน้าเว็บ (กด F5 หรือกดโหลดซ้ำ)
3. คุณจะพบเมนูพิเศษชื่อ **"⚙️ VTuber Gateway Config"** ปรากฏขึ้นที่แถบเครื่องมือด้านบนสุด (ข้าง ๆ เมนูช่วยเหลือ) ให้ทำการคลิกเลือกเมนูย่อย **🔑 Configure Keys & Secrets**
4. ในการรันครั้งแรก Google จะขึ้นหน้าต่างเตือนสิทธิ์ความปลอดภัย (Authorization Required) ให้คลิก **Continue (ดำเนินการต่อ)** -> เลือกบัญชี Google ของคุณ -> คลิกหัวข้อ **Advanced (ขั้นสูง)** -> คลิกเลือก **Go to ... (ปลอดภัย)** -> กดปุ่ม **Allow (อนุญาต)** 
5. เมื่ออนุญาตสิทธิ์เรียบร้อยแล้ว แผงควบคุมดีไซน์เรียบหรูจะสไลด์เปิดขึ้นที่ด้านขวาของหน้าต่างทันทีค่ะ ให้กรอกข้อมูลที่คุณคัดลอกมาดังนี้:
   * **FIREBASE DATABASE URL:** วางลิงก์ฐานข้อมูลที่คัดลอกจาก Firebase (เช่น `https://your-project.firebasedatabase.app/`)
   * **FIREBASE SECRET KEY:** วางรหัสลับฐานข้อมูลที่ได้จากแท็บบัญชีบริการ Service Accounts
   * **XENDIT SECRET KEY:** วางรหัส API Key ที่ขึ้นต้นด้วย `xnd_...`
   * **STREAMLABS ACCESS TOKEN:** วางรหัสโทเค็นถาวรสำหรับส่งข้อมูลเด้งสตรีม (หากไม่ใช้ ให้ปล่อยว่างได้ค่ะ)
   * **WEBHOOK VERIFICATION_TOKEN:** วางคีย์ Webhook Verification Token ที่ได้จากหน้ารายละเอียด Webhooks ของ Xendit
6. กดปุ่ม **Save Configuration** ด้านล่างสุด ระบบหลังบ้านจะจัดระเบียบและทำการเซฟค่าความปลอดภัยทั้งหมดลงสคริปต์ให้โดยอัติโนมัติอย่างปลอดภัยค่ะ!

### ขั้นตอนที่ 2.5: การขึ้นระบบใช้งานจริงในฐานะเว็บแอป (Deploy Web App)
นี่คือขั้นตอนสุดท้ายในการเผยแพร่หน้าต่างการโอนเงินออกสู่สาธารณะค่ะ

1. ในหน้าจอเขียนโปรแกรม Apps Script (จากขั้นตอนที่ 2.1) ให้คลิกปุ่ม **Deploy (การทำให้ใช้งานได้)** ที่มุมขวาบน -> เลือก **New Deployment (การทำให้ใช้งานได้ใหม่)**
2. คลิกรูปเฟืองข้างคำว่า "Select Type" แล้วเลือกประเภทเป้าหมายเป็น **Web App (เว็บแอป)**
3. กำหนดรายละเอียดสิทธิ์ดังนี้:
   * **Description (รายละเอียด):** เช่น `VTuber Secure Tips v1.0`
   * **Execute as (เรียกใช้งานในฐานะ):** เลือกเป็น **Me (บัญชีของคุณเอง)** เพื่อให้หลังบ้านมีสิทธิ์ในการประมวลผลข้อมูลฐานข้อมูลให้ผู้ใช้ค่ะ
   * **Who has access (ผู้มีสิทธิ์เข้าใช้งาน):** เลือกเป็น **Anyone (ทุกคน)** เพื่อให้ผู้บริจาคทั่วไปสามารถเรียกเปิดหน้ารับบริจาคนี้ได้
4. กดปุ่ม **Deploy**
5. รอให้ระบบประมวลผลสักครู่ คุณจะได้รับข้อความแจ้งสถานะสำเร็จ และจะพบคีย์ลิงก์ในหัวข้อ **Web App URL** ให้ทำการคัดลอกลิงก์นี้เก็บไว้เพื่อนำไปแชร์ให้ผู้ชมของคุณ หรือนำไปใส่ในโปรแกรมสตรีมเมอร์ค่ะ!

### ขั้นตอนที่ 2.6: การเชื่อม Webhook สัญญาณการจ่ายเงินจาก Xendit เข้าหาเกตเวย์
เพื่อแจ้งยอดชำระเงินแบบเรียลไทม์ เราต้องผูกลิงก์ Web App ที่ได้จากขั้นตอนที่ 2.5 เข้าหาบัญชี Xendit ของคุณค่ะ

### ตัวอย่างการปรับแต่งที่ 4: การขยับยอดบริจาคขั้นต่ำเพื่อป้องกันบ็อตสแปม 1 บาท (Min-Max Limits)
ในสถิติบางครั้งอาจมีผู้ใช้งานสแปมปุ่มโอนเงินจำนวน 1 บาทรัว ๆ ส่งผลให้ยอดการตัดรอบและการยิงสัญญาณแจ้งเตือนก่อความรำคาญใจ คุณสามารถปรับให้ผู้ชมต้องโอนขั้นต่ำเริ่มต้นตั้งแต่ 10 บาทขึ้นไปได้ง่าย ๆ ดังนี้ค่ะ:

*   **ไฟล์ที่ต้องแก้ไข 1:** `src/PaymentGateway.gs` (ระบบกรองหลังบ้าน)
    *   **จุดที่ต้องค้นหา:** บรรทัดตรวจสอบเงื่อนไขความถูกต้อง
        ```javascript
        if (isNaN(amount) || amount < 1.00 || amount > 700000.00) return Utils.createValidationError("ยอดสนับสนุนที่กำหนดได้ ต้องอยู่ระหว่าง 1.00 ถึง 700,000.00 บาทค่ะ");
        ```
    *   **ตัวอย่างการปรับยอดขั้นต่ำเป็น 10 บาท และเพดานสูงสุดไม่เกิน 20,000 บาท:**
        ```javascript
        if (isNaN(amount) || amount < 10.00 || amount > 20000.00) return Utils.createValidationError("ยอดสนับสนุนที่กำหนดได้ ต้องอยู่ระหว่าง 10.00 ถึง 20,000.00 บาทค่ะ");
        ```
*   **ไฟล์ที่ต้องแก้ไข 2:** `src/GrapesJSForm.html` (เงื่อนไขดักฝั่งหน้าเว็บผู้ใช้งาน)
    *   **จุดที่ต้องค้นหา:** ฟิลด์กรอกตัวเลขยอดเงินบริจาค
        ```html
        <input type="number" id="amount" name="amount" required min="1.00" max="700000.00" step="0.01" class="gjs-form-input" style="padding-left: 28px; padding-right: 48px;" placeholder="20.00" aria-required="true">
        ```
    *   **ตัวอย่างการแก้ให้ล้อตามระบบตรวจสอบหลังบ้าน:** ปรับแก้ในแท็กคุณลักษณะ `min` และ `max` รวมถึง `placeholder` ดังนี้ค่ะ:
        ```html
        <input type="number" id="amount" name="amount" required min="10.00" max="20000.00" step="0.01" class="gjs-form-input" style="padding-left: 28px; padding-right: 48px;" placeholder="10.00" aria-required="true">
        ```
