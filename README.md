# VTuber Secure Tips (PDPA-Compliant Model)

## 1. ภาพรวมโครงการและสถาปัตยกรรม (Project Overview & Architecture)

### สรุปการทำงานของระบบ
**VTuber Secure Tips** คือระบบ Gateway สำหรับรับเงินสนับสนุน (Donation) สำหรับ Vtuber โดยเน้นความปลอดภัยระดับสูงและการปฏิบัติตามกฎหมายคุ้มครองข้อมูลส่วนบุคคล (PDPA) ระบบทำหน้าที่เป็นตัวกลางในการสร้าง PromptPay QR Code ผ่าน Xendit, บันทึกประวัติการโอนเงินลง Firebase Realtime Database และส่งการแจ้งเตือนไปยัง Streamlabs เพื่อให้ปรากฏบนหน้าจอไลฟ์สตรีมแบบ Real-time

### เทคโนโลยีที่ใช้ (Tech Stack)
- **Backend Runtime:** Google Apps Script (GAS) - V8 Engine
- **Frontend:** HTML5, CSS3 (Modern CSS with Glassmorphism), JavaScript (ES6+)
- **Database:** Firebase Realtime Database (NoSQL)
- **Payment Gateway:** Xendit API (PromptPay Dynamic QR)
- **Alert System:** Streamlabs API
- **Security:**
    - **Proof-of-Work (PoW):** ป้องกัน Bot/Spam ในระดับ Client-side
    - **HMAC Session Signing:** ตรวจสอบความถูกต้องของเซสชันด้วย Cryptographic Signature
    - **Circuit Breaker Pattern:** ป้องกันระบบล่มเมื่อ API ภายนอก (Xendit/Streamlabs) มีปัญหา
    - **CSP (Content Security Policy):** ป้องกัน XSS และ Clickjacking

### รูปแบบสถาปัตยกรรม (Architectural Patterns)
ระบบใช้รูปแบบ **Layered Architecture** เพื่อแยกความรับผิดชอบ (Separation of Concerns):
1.  **Routing Layer (`Router.gs`):** จัดการ Request (doGet/doPost) และการ Render หน้าจอ
2.  **Service Layer (`Security.gs`, `DatabaseService.gs`, `PaymentGateway.gs`, `AlertService.gs`):** จัดการ Logic ทางธุรกิจและการประมวลผลหลัก
3.  **Infrastructure Layer (`Config.gs`, `Utils.gs`, `CircuitBreaker.gs`):** จัดการการตั้งค่า, ตัวช่วยทั่วไป และการควบคุมความเสถียรของระบบ

## 2. การไหลของข้อมูลและโครงสร้างฐานข้อมูล (Data Flow & Schema)

### การไหลของข้อมูล (Data Flow)
1.  **Session Handshake:** User $\rightarrow$ `Index.html` (Solve PoW) $\rightarrow$ `Security.initializeSession` $\rightarrow$ Receive Signed Session Token.
2.  **Payment Request:** User $\rightarrow$ `GrapesJSForm.html` $\rightarrow$ `PaymentGateway.createPayment` (Verify Token) $\rightarrow$ Xendit API $\rightarrow$ Return QR String.
3.  **Payment Confirmation:** Xendit Webhook $\rightarrow$ `Router.doPost` $\rightarrow$ `PaymentGateway.verifyPaymentDirectlyWithXendit` (Double Check) $\rightarrow$ Write to Firebase $\rightarrow$ Send to Streamlabs.

## 🚀 จุดเด่นทางระบบความปลอดภัยและฟีเจอร์เด่น
1. **สอดคล้องกฎหมาย PDPA/GDPR:** ไม่เก็บข้อมูลชื่อบัญชีจริงหรือเลขบัญชีธนาคาร แสดงผลเฉพาะชื่อเล่นที่ผู้ใช้กรอกเข้ามาเท่านั้น
2. **ระบบสับคัตเอาต์เยียวยาตัวเองอัตโนมัติ (Circuit Breaker):** หลีกเลี่ยงหน้าสตรีมค้างหรือล่ม โดยระบบจะตัดสัญญาณฝั่งหน้าจอเตือน Streamlabs อัตโนมัติหากพบสัญญาณติดขัด แต่กระบวนการชำระเงินยังทำงานปกติ
3. **ระบบสกัดบ็อตก่อกวน (Anti-Automation Throttling):** กลไกบังคับฝั่งหน้าเว็บให้คำนวณถอดโจทย์พรูฟออฟเวิร์ก (Proof-of-Work) ร่วมกับมาตรการบล็อกยอดคำขอพุ่งกระฉูดระดับ Global Spike
4. **นโยบายความมั่นคงปลอดภัยสคริปต์ (Strict Nonce CSP):** ป้องกันช่องทางลักลอบโจมตีหรือเปลี่ยนโครงสร้างโค้ดหน้าเว็บบนเบราว์เซอร์ด้วยสัญชาตลักษณ์ป้องกันสคริปต์แอบแฝง
5. **ระบบทุบทำลายประวัติธุรกรรมประจำวัน (30-Day Data Purging):** ติดตั้งระบบทำงานนัดแนะตรวจสอบประวัติในฐานข้อมูล Firebase ทุก 24 ชั่วโมงเพื่อลบข้อมูลที่อายุเกิน 30 วันทิ้งตามกฎหมายกำหนด

## 2. ขั้นตอนการติดตั้งอย่างละเอียด (Step-by-Step Installation)

### ขั้นตอนที่ 2.1: การสร้างและวางโค้ดใน Google Sheets
สามารถ copy sheet นี้ https://docs.google.com/spreadsheets/d/1VkZsCaL6c5HldQrHj8ESVkM5VwLxRWifzLP5KQJL1Sk/copy เพื่อทำสำเนา code ไปใช้งานได้เลย

#### 1. ทำสำเนาระบบเข้าสู่สเปรดชีตส่วนตัว
* คลิกเปิดลิงก์ตัวช่วยติดตั้งระบบที่คุณได้รับ (ลิงก์ที่ลงท้ายด้วย `/copy`)
* หน้าจอของ Google จะแสดงกล่องข้อความถามยืนยัน ให้คลิกปุ่ม **"ทำสำเนา (Make a copy)"**
* ระบบจะทำการสร้างชีตใหม่พร้อมจำลองโค้ดรักษาความปลอดภัยและหน้าจอดีไซน์ทั้งหมดมาเก็บไว้ในไดรฟ์ส่วนตัวของคุณให้โดยอัตโนมัติ

#### 2. สมัครใช้คลังข้อมูลเก็บประวัติความปลอดภัย Firebase
* เปิดเบราว์เซอร์ไปที่ [Firebase Console](https://console.firebase.google.com/) แล้วล็อกอินด้วยบัญชี Google ของคุณ
* คลิกปุ่ม **Add project (เพิ่มโครงการ)** -> ตั้งชื่อโครงการของคุณ เช่น `my-secure-donation` -> กดถัดไปและเปิดสิทธิ์จนเสร็จเรียบร้อย (แนะนำให้กดปิดตัวระบบ Google Analytics เพื่อให้ติดตั้งเสร็จไวขึ้น)
* ที่เมนูด้านซ้าย เลือก **Build (สร้าง)** -> **Realtime Database** -> คลิก **Create Database (สร้างฐานข้อมูล)**
* ให้ระบุเลือกประเทศเซิร์ฟเวอร์เป็น **Singapore (asia-southeast1)** และคลิกเลือกโหมดความปลอดภัย **Locked mode (โหมดล็อก)** แล้วกดเปิดใช้งาน
* สลับไปที่แท็บ **Rules (กฎ)** ด้านบน -> คัดลอกโค้ดตรวจสอบสิทธิ์จากกล่องด้านล่างนี้ไปวางทับค่าเดิมทั้งหมด แล้วกดปุ่ม **Publish (เผยแพร่)**:
  ```json
  {
    "rules": {
      "donations": {
        ".indexOn": ["created_at"],
        "$donation_id": {
          ".read": "auth != null",
          ".write": "auth != null",
          ".validate": "newData.hasChildren(['id', 'nickname', 'amount', 'status', 'created_at']) && newData.child('amount').isNumber() && newData.child('amount').val() >= 1"
        }
      }
    }
  }
  ```
* กดกลับมาที่แท็บ **Data (ข้อมูล)** แล้วทำการคัดลอกลิงก์ที่ขึ้นต้นด้วย `https://` บนหน้าจอกล่องข้อมูลเก็บไว้
* **วิธีดึงรหัส Database Secret:** คลิกสัญลักษณ์รูปฟันเฟือง ⚙️ ด้านซ้ายข้างเมนู Project Overview -> เลือก **Project settings (การตั้งค่าโครงการ)** -> คลิกแท็บ **Service accounts (บัญชีบริการ)** ด้านบน -> คลิกหัวข้อ **Database secrets (รหัสลับฐานข้อมูล)** -> กดแสดงสิทธิ์รหัสลับของโครงการของคุณแล้วกดปุ่มคัดลอกเก็บไว้

#### 3. สมัครใช้บริการแปลงธุรกรรม PromptPay ผ่าน Xendit
* สมัครและล็อกอินที่หน้า [Xendit Dashboard](https://dashboard.xendit.co/) (เปิดโหมด Test Mode ไว้ก่อนเพื่อใช้จำลองสแกนเงินสมมติ)
* ไปที่เมนู **Settings (การตั้งค่า)** -> หัวข้อย่อยด้านล่างชื่อ **API Keys** -> กดสร้างรหัสโดยเลือกตั้งค่าหัวข้อ **Money-in Products** ให้เป็นแบบ **Write (เขียน)** และหัวข้ออื่นเป็น None ทั้งหมด
* ยืนยันระบบแล้วคัดลอกตัวรหัส API Key ที่ขึ้นต้นด้วยข้อความ `xnd_live_...` หรือ `xnd_test_...` เก็บไว้
* ไปที่เมนู **Settings** อีกครั้ง -> เลื่อนลงไปหัวข้อย่อยชื่อ **Webhooks** -> เลื่อนหน้าจอลงมาด้านล่างสุดจะพบรหัสยาว ๆ ในช่อง **Webhook Verification Token** ให้กดคัดลอกเก็บไว้

#### 4. ป้อนข้อมูลระบบและติดตั้ง Web App
* สลับกลับมาที่ชีตที่คุณทำสำเนาไว้ในขั้นตอนที่ 1 
* กดปุ่มรีเฟรชหน้าเว็บ (F5) หนึ่งครั้ง จะมีเมนูพิเศษชื่อ **"⚙️ VTuber Gateway Config"** โผล่ขึ้นมาข้าง ๆ ปุ่มความช่วยเหลือด้านบนสุด
* ให้คลิกปุ่ม **⚙️ VTuber Gateway Config** -> เลือก **🔑 Configure Keys & Secrets**
* ในการเปิดใช้ครั้งแรก ระบบความปลอดภัยของ Google จะขึ้นเตือนเรื่องการยินยอมการขอสิทธิ์เข้าถึงสเปรดชีต ให้กด **Continue** -> เลือกอีเมลของคุณ -> กดคำว่า **Advanced (ขั้นสูง)** -> กดปุ่มสีเทาด้านล่างสุด **Go to... (ปลอดภัย)** -> กดปุ่ม **Allow (อนุญาต)** เพื่อรับรองสิทธิ์ความปลอดภัย
* เมื่อยืนยันสิทธิ์เสร็จแล้ว ให้คลิกเลือกปุ่ม **Configure Keys & Secrets** อีกครั้ง แผงควบคุมสไลด์บาร์จะโผล่ขึ้นมาที่ขวามือของชีต ให้กรอกรหัสข้อมูลที่คุณเตรียมไว้ทั้งหมดลงไปในแต่ละช่องให้ถูกต้อง และกดปุ่ม **Save Configuration**
* จากนั้นไปที่หน้าเขียนโค้ดหลังบ้าน (Extensions -> Apps Script) -> กดปุ่ม **Deploy (การทำให้ใช้งานได้)** ที่มุมขวาบน -> เลือก **New Deployment** -> กดเลือกประเภทเป้าหมายรูปเฟืองเป็น **Web App (เว็บแอป)** -> เลือก Execute as: **Me** และ Who has access: **Anyone** แล้วกดปุ่ม Deploy
* คัดลอกลิงก์ยาว ๆ ที่ระบบให้มาในช่อง **Web App URL**
* นำลิงก์ Web App URL ดังกล่าว ย้อนกลับไปใส่ลงในหน้าตั้งค่าเว็บฮุกของ Xendit ([Settings -> Webhooks](https://dashboard.xendit.co/)) ในหัวข้อพฤติกรรมธุรกรรมชำระเงินชื่อ **QR code paid** แล้วกดปุ่ม Test and save เป็นอันเสร็จสิ้นพิธีติดตั้งระบบ

## 🎨 คู่มือปรับปรุงและตกแต่งหน้าจอโดยใช้เครื่องมือ GrapesJS 

### ขั้นตอนที่ 1: การเตรียมโค้ดหน้าตาเดิมที่คุณต้องการตกแต่ง
คุณสามารถเลือกตกแต่งได้ 2 หน้าจอหลักตามไฟล์ที่มีในระบบ:
*   **หากต้องการแต่งหน้า "กรอกข้อมูลบริจาค" (หน้าแรกสุด):** ให้ไปที่โครงการหลังบ้านใน Google Sheets แล้วคัดลอกโค้ดทั้งหมดจากไฟล์ **`src/GrapesJSForm.html`**
*   **หากต้องการแต่งหน้า "แสดง PromptPay QR Code" (หน้าหลังโอนเงิน):** ให้คัดลอกโค้ดทั้งหมดจากไฟล์ **`src/GrapesJSQR.html`**

### ขั้นตอนที่ 2: การเข้าสู่โปรแกรมแต่งรูปภาพและนำเข้าโค้ด (Import)
1. เปิดหน้าเว็บเบราว์เซอร์และเข้าไปที่: [https://app.grapesjs.com/studio](https://app.grapesjs.com/studio)
2. คุณจะพบบิวเดอร์หน้าเปล่าที่มีเครื่องมือจัดวางอยู่รอบด้าน ให้สังเกตที่แถบเครื่องมือด้านบนหรือด้านข้าง จะมีปุ่ม **"Import Code"** ปรากฏอยู่
3. คลิกที่ปุ่มดังกล่าว หน้าเว็บจะเปิดกล่องข้อความสีเข้มขึ้นมาให้คุณป้อนโค้ด
4. ให้ทำการ **วาง (Paste)** โค้ดที่คุณคัดลอกเตรียมไว้จากขั้นตอนที่ 1 ลงไปในกล่องข้อความนั้นได้เลยค่ะ
5. กดปุ่ม **"Import"** หรือ **"Apply"** ด้านล่าง หน้าจอจำลองการ์ดโอนเงินเวอร์ชันภาษาไทยดั้งเดิมจะถูกเรนเดอร์ขึ้นมาบนแคนวาสให้คุณเลือกคลิกตกแต่งได้ทันที

### ขั้นตอนที่ 3: ลงมือแต่งหน้าจอให้สวยงามด้วยแผงควบคุมอัจฉริยะ
คุณสามารถคลิกเลือกชิ้นส่วนต่าง ๆ บนหน้าจอเพื่อปรับแต่งได้ตามความสร้างสรรค์ของคุณ

#### 🎨 วิธีการเปลี่ยนโทนสีพื้นหลังและการ์ดหลัก:
*   ให้คลิกที่พื้นที่พื้นหลังด้านนอกสุด หรือคลิกตรงตัวการ์ดรับเงิน
*   ดูที่แผงควบคุมสไตล์ด้านขวา (Style Manager) หัวข้อ **Background (พื้นหลัง)** หรือ **Color (สี)**
*   คุณสามารถคลิกเลือกเปลี่ยนเฉดสีเดี่ยว ๆ หรือจะเลือกทำเป็นสีไล่เฉด (Gradient) สุดหรูหราสไตล์แฟนตาซีก็สามารถเลือกจิ้มบนแผงควบคุมได้เลย

#### ✍️ วิธีการแก้ไขตัวอักษร ฟอนต์ และขนาดข้อความ:
*   ดับเบิ้ลคลิก (Double-click) ไปที่ตัวอักษรใด ๆ บนหน้าจอ (เช่น หัวข้อ "VTuber Secure Tips") เพื่อแก้ไขคำพูด พิมพ์ทับคำใหม่ หรือปรับเปลี่ยนสไตล์ตัวเอียง/ตัวหนาได้ทันที
*   ที่แผงสไตล์ด้านขวา หัวข้อ **Typography (การจัดหน้าอักษร)** คุณสามารถเลือกเปลี่ยนขนาดตัวอักษร (Font Size), ระยะห่างระหว่างบรรทัด, หรือแม้กระทั่งความหนาบางของตัวอักษรได้ตามต้องการ

#### 📏 วิธีการปรับระยะเยื้อง ขยับพิกัด และความสูง (Spacing & Borders):
*   หากต้องการขยับปุ่มกด หรือปรับการ์ดให้มีความโค้งมนละมุนขึ้น ให้คลิกที่การ์ดแล้วเลื่อนไปที่แผงด้านขวาหัวข้อ **Decorations (การตกแต่ง)** -> **Border Radius (ความโค้งมนของขอบ)** แล้วลองปรับเปลี่ยนตัวเลข
*   คุณสามารถลากแถบขอบรอบชิ้นส่วนเพื่อเพิ่มระยะห่าง (Margin / Padding) ได้อย่างอิสระ

> ### ⚠️ กฎความปลอดภัยเหล็กที่ห้ามลืมเด็ดขาด! (Crucial Alert)
> แม้ว่าคุณจะสามารถปรับเปลี่ยนขนาด สีสัน และสไตล์ของแบบฟอร์มได้อย่างไร้ขีดจำกัด **แต่ห้ามลบ ย้าย หรือแก้ไข "รหัสเบื้องหลัง (Attributes)" ของชิ้นส่วนหลักเหล่านี้เด็ดขาดนะคะ:**
> *   ห้ามแก้ไขค่าในกล่อง **ID** และ **Name** ของช่องรับข้อมูลเดิม (สามารถดูชื่อ ID ได้โดยการกดเลือก Properties) เช่น ช่องชื่อเล่น (`id="nickname"`), ช่องข้อความสนับสนุน (`id="message"`), ช่องจำนวนเงิน (`id="amount"`), และกล่องยอมรับข้อมูลส่วนตัว (`id="pdpa-consent"`) 
> *   ห้ามแก้ไขพื้นที่วางภาพคิวอาร์โค้ด ซึ่งทำงานผ่านกล่องรหัสสากลชื่อ **`id="qr-code-container"`**
> *   *เหตุผล:* หากรหัสระบุตัวตนเหล่านี้หายไป ระบบหลังบ้านของ Google Apps Script จะตรวจจับฟิลด์รับข้อมูลไม่เจอ และจะทำให้ระบบไม่สามารถผลิตภาพคิวอาร์โค้ดส่งกลับมาให้คุณได้

### ขั้นตอนที่ 4: คัดลอกโค้ดใหม่กลับไปเซฟใช้งานจริง (Export)
เมื่อคุณทำการลากปรับขนาด ขยับตำแหน่ง ปรับเปลี่ยนชุดสีพาสเทลจนเป็นที่น่าพอใจแล้ว ให้ทำตามวิธีดึงโค้ดเวอร์ชันอัปเกรดกลับมาดังนี้:

1. คลิกที่ปุ่มปุ่ม **"Import Code"** ที่ด้านขวาบนของโปรแกรม GrapesJS
2. ระบบจะทำการรวบรวม HTML และ CSS ออกมาเป็นโค้ดชุดใหม่ที่สะอาด เรียบร้อย และสวยงาม
3. ให้ทำการคัดลอกโค้ดทั้งหมดที่ปรากฏอยู่ในช่องแสดงผลนั้นไว้
4. สลับหน้าจอเปิดกลับไปที่ Apps Script ในโครงการ Google Sheets ของคุณ 
5. เข้าไปที่ไฟล์ที่คุณดึงมาตอนแรกสุด (เช่น หากก่อนหน้านี้แต่งหน้าจอแรก ให้เปิดไฟล์ `src/GrapesJSForm.html`)
6. ลบโค้ดเก่าในไฟล์นั้นทิ้งไปให้หมด แล้วทำการ **วาง (Paste)** โค้ดชุดแต่งใหม่ลงไปแทนที่
7. กดบันทึก (กด Ctrl + S หรือสัญลักษณ์รูปแผ่นดิสก์)
8. **เสร็จเรียบร้อย**

สำหรับครีเอเตอร์ที่อยากปรับแก้ข้อความ เปลี่ยนยอดปุ่มกดบริจาคด่วน หรือเสริมหน้าตาอนิเมะของตนเองลงในหน้าจอโอนเงินให้เข้ากับเอกลักษณ์ของช่อง สามารถเลื่อนไปหาจุดแก้ไขในไฟล์ HTML หน้าจอได้ ดังนี้: (หรือสามารถใช้เครื่องมือ https://app.grapesjs.com/studio เพื่อตกแต่งเบื้องต้นได้)

### 1. วิธีปรับแต่งขอบเขต ยอดโอนเงินสมทบขั้นต่ำสุด และขั้นสูงสุด
หากต้องการเปลี่ยนเกณฑ์ยอดโอนเงินขั้นต่ำเริ่มต้นตั้งแต่ 5 บาทขึ้นไป และจำกัดไม่ให้สแปมยอดการชำระเงินในโหมดสแกนเกิน 5,000 บาท

*   **จุดที่ 1 (ตัวดักฝั่งหน้าเว็บ):** ให้เปิดไฟล์ `src/GrapesJSForm.html` แล้วค้นหาคำว่า `id="amount"` แล้วทำการแก้ไขช่วงตัวเลขที่แท็ก `min` และ `max` ดังนี้:
    ```html
    <!-- เดิม min="1.00" max="700000.00" แก้ไขเป็นยอดใหม่ที่เราต้องการดักหน้าจอ -->
    <input type="number" id="amount" name="amount" required min="5.00" max="5000.00" step="0.01" class="gjs-form-input" style="padding-left: 28px; padding-right: 48px;" placeholder="5.00" aria-required="true">
    ```
*   **จุดที่ 2 (ตัวป้องกันฝั่งหลังบ้าน):** เปิดไฟล์ `src/PaymentGateway.gs` แล้วค้นหารหัสที่ทำหน้าที่ปัดเป่าปริมาณเงินเกินขอบเขต:
    ```javascript
    // เดิม: isNaN(amount) || amount < 1.00 || amount > 700000.00
    // ให้เปลี่ยนช่วงเงินขั้นต่ำและเพดานความปลอดภัยให้ล้อตรงกันกับหน้าบ้านค่ะ:
    if (isNaN(amount) || amount < 5.00 || amount > 5000.00) return Utils.createValidationError("ยอดสนับสนุนที่กำหนดได้ ต้องอยู่ระหว่าง 5.00 ถึง 5,000.00 บาทค่ะ");
    ```

### 2. วิธีการเปลี่ยนค่าในปุ่มยอดเงินสนับสนุนด่วน (Presets)
คุณสามารถแก้ไขจำนวนเงินด่วนในปุ่มเพื่อให้ผู้ชมที่ต้องการส่งยอดไว ๆ กดคลิกและข้ามไปหน้าสแกน QR ได้ง่ายขึ้น

*   **จุดแก้ไข:** เปิดไฟล์ `src/GrapesJSForm.html` ค้นหาคำว่า `class="gjs-form-presets"`
*   **ตัวอย่างการแต่งพรีเซ็ตตามใจชอบ (เช่น ยอด 10, 50, 100, 1000 บาท):**
    ให้แก้ข้อมูลในสัญชาตลักษณ์ `data-preset` และตัวอักษรไทยหน้าปุ่มให้สอดคล้องกันดังนี้:
    ```html
    <div class="gjs-form-presets">
      <button type="button" data-preset="10" class="gjs-form-preset-btn">฿10</button>
      <button type="button" data-preset="50" class="gjs-form-preset-btn">฿50</button>
      <button type="button" data-preset="100" class="gjs-form-preset-btn">฿100</button>
      <button type="button" data-preset="1000" class="gjs-form-preset-btn">฿1,000</button>
    </div>
    ```

## 🛠️ คู่มือการนำหน้าเว็บไปฝังบน Google Sites หากต้องการปรับแต่ง URL ให้มีชื่อช่อง

### ขั้นตอนที่ 1: คัดลอกลิงก์ Web App
*   เข้าไปที่โครงการหลังบ้านในหน้า Apps Script
*   กดปุ่ม **Deploy (การทำให้ใช้งานได้)** -> เลือก **Manage Deployments (จัดการการทำให้ใช้งานได้)**
*   คัดลอกลิงก์ที่อยู่ในช่อง **Web App URL** (ลิงก์นี้จะขึ้นต้นด้วย `https://script.google.com/macros/s/...`)

### ขั้นตอนที่ 2: เปิดโปรแกรมแก้ไข Google Sites
*   เปิดเบราว์เซอร์ไปที่ [Google Sites](https://sites.google.com/) แล้วเปิดหน้าเว็บที่คุณต้องการแก้ไข หรือสร้างไซต์ใหม่ขึ้นมา
*   ไปที่หน้าที่คุณต้องการวางแบบฟอร์มรับเงิน (เช่น สร้างหน้าใหม่ชื่อ "สนับสนุนช่อง / Donate")

### ขั้นตอนที่ 3: แทรกหน้าต่างแบบฝังตัว (Embed Web App)
*   ที่แถบเครื่องมือด้านขวา คลิกเลือกแท็บ **แทรก (Insert)** -> เลือกเมนู **ฝัง (Embed)**
*   จะปรากฏหน้าต่างป๊อปอัปขึ้นมา ให้เลือกแท็บ **"ด้วย URL (By URL)"**
*   **วาง (Paste)** ลิงก์ Web App URL ที่คุณคัดลอกมาจากขั้นตอนที่ 1 ลงไปในช่องป้อนข้อมูล
*   ระบบจะทำการสแกนตรวจสอบหน้าจอชั่วครู่ ให้คุณคลิกเลือกหัวข้อ **"ทั้งหน้า (Whole page)"** เพื่อให้แสดงผลแบบไร้รอยต่อเสมือนเป็นเว็บไซต์เดียวกัน
*   คลิกปุ่ม **แทรก (Insert / Add)** สีน้ำเงิน

### ขั้นตอนที่ 4: ขยายขนาดกรอบหน้าจอให้พอดี (สำคัญมาก!)
*   หน้าต่างกรอกแบบฟอร์มบริจาคสีเข้มจะปรากฏขึ้นมาบนหน้า Google Sites ของคุณ
*   ให้ใช้เมาส์คลิกจับที่ **"จุดกลมสีน้ำเงินที่ขอบด้านล่างและด้านข้าง"** ของกรอบไอเฟรม (Iframe Container) แล้วทำการลากขยายขอบลงมาด้านล่างให้ยาวที่สุดเท่าที่จะทำได้
*   *จุดประสงค์:* ขยายความสูงของกรอบให้ครอบคลุมขนาดของการ์ดกรอกเงินบริจาค เพื่อป้องกันไม่ให้ผู้ชมสตรีมต้องคอยเลื่อนปุ่ม Scrollbar ด้านข้างขณะเปิดดูผ่านมือถือ ซึ่งจะทำให้หน้าเว็บดูสวยงาม ไร้รอยต่อ และเป็นเนื้อเดียวกันอย่างสมบูรณ์

### ขั้นตอนที่ 5: กดเผยแพร่เว็บไซต์ (Publish)
*   กดปุ่ม **เผยแพร่ (Publish)** ที่มุมขวาบนของ Google Sites
*   คุณจะได้ลิงก์เป็นโดเมนของ Google Sites ที่สวยงามและสามารถนำไปแชร์ให้ผู้ชมสแกนโอนเงินได้อย่างปลอดภัย

## 💡 คำแนะนำเพิ่มเติมเพื่อสิทธิ์การใช้งานที่ราบรื่น (Pro-Tips)

*   **ตั้งค่าสิทธิ์การเข้าถึง Web App ให้เหมาะสม:** ในขั้นตอนการ Deploy Web App ในหน้า Apps Script ของคุณ กรุณาตรวจสอบให้แน่ใจเสมอนะคะว่าได้ตั้งค่าหัวข้อ **"Who has access"** เป็น **"Anyone" (ทุกคน)** แล้ว เพื่อให้ผู้ชมสตรีมที่เปิดเข้ามาดูบน Google Sites ทุกคนสามารถมองเห็นและกรอกแบบฟอร์มส่งยอดได้โดยไม่ต้องล็อกอินบัญชี Google
